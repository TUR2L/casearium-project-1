# Задача 2: B2C консультант (RAG Public)

**Исполнитель:** Павел  
**Инструмент:** Evo Managed RAG  
**Приоритет прототипа:** Высокий — обрабатывает наибольший объём входящих обращений

---

## Роль в системе

Павел отвечает за **агента-консультанта для физических лиц** — второй узел цепочки. Агент активируется после классификатора (Задача 1), когда определён тип клиента B2C и категория `consultation`.

Это самый высоконагруженный модуль: большинство обращений физлиц — типовые вопросы по тарифам, переводам и работе приложения. Агент должен давать точные ответы из базы знаний без привлечения человека.

---

## Что нужно построить

RAG-агент с подключённой публичной базой знаний, который отвечает клиенту на естественном языке в дружелюбном тоне.

### Входные данные

```json
{
  "client_type": "B2C",
  "category": "consultation",
  "message": "Какая комиссия за перевод между своими счетами?",
  "dialog_history": []
}
```

### Выходные данные

```json
{
  "answer": "Переводы между вашими собственными счетами в нашем банке — бесплатны и выполняются мгновенно. Если вас интересует перевод на счёт в другом банке, уточните, пожалуйста, — расскажу подробнее.",
  "answer_found": true,
  "source": "faq_transfers",
  "escalate_to_human": false
}
```

## Источники знаний (прототип)

Для прототипа используются вымышленные документы. Можно создать моки:

| Документ | Содержание |
|----------|------------|
| `faq_transfers.md` | Вопросы и ответы о переводах, лимитах, комиссиях |
| `faq_cards.md` | Вопросы о картах: выпуск, блокировка, лимиты |
| `faq_app.md` | Работа мобильного приложения, вход, восстановление |
| `pricing.md` | Тарифы и условия обслуживания для физлиц |

## Логика работы

### Семантический поиск (RAG)

1. Получить вопрос клиента
2. Найти релевантные фрагменты в базе знаний (top-3 чанка)
3. Сгенерировать ответ на основе найденных фрагментов
4. Если релевантных чанков нет — установить `answer_found: false` и эскалировать

### Tone of Voice

Агент общается как **дружелюбный и грамотный специалист поддержки**:

| Правильно | Неправильно |
|-----------|-------------|
| «Переводы между вашими счетами бесплатны» | «Согласно пункту 3.2 тарифного плана...» |
| «Уточните, пожалуйста, какой именно перевод вас интересует?» | «Нет данных» |
| «Рад помочь! Вот как это работает...» | «Читайте FAQ на сайте» |

### Эскалация на человека

Переводить на человека при:
- `answer_found: false` — вопрос не найден в базе знаний
- Прямой просьбе клиента: «соедините с оператором», «хочу говорить с человеком»
- Более 2 уточняющих вопросов без достижения ответа

## Тестовые сценарии

| Запрос | Ожидаемое поведение |
|--------|---------------------|
| «Какая комиссия за перевод между своими счетами?» | Ответ из FAQ, `answer_found: true` |
| «Как заблокировать карту?» | Ответ из FAQ по картам |
| «Что такое кешбэк?» | Ответ из тарифов |
| «Расскажи про квантовые инвестиции» | `answer_found: false`, эскалация |
| «Хватит, соедините с человеком» | `escalate_to_human: true` |

---

## Критерий приёма прототипа

- Агент отвечает на вопрос «Какая комиссия за перевод между своими счетами?» из тестового демо
- При отсутствии ответа в базе знаний корректно эскалирует, а не выдумывает ответ
- Ответы написаны в дружелюбном тоне, без технического жаргона
