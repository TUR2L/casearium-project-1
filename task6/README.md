# Задача 6: Обработчик жалоб (Complaints & ToV)

**Исполнитель:** Арима  
**Инструмент:** Evo Agents  
**Приоритет прототипа:** Высокий — работа с негативом напрямую влияет на NPS

---

## Роль в системе

Арима отвечает за **агента обработки жалоб** — один из самых деликатных модулей системы. Агент активируется, когда классификатор определил категорию `complaint`.

Именно здесь происходит работа с недовольными клиентами: важно не только решить проблему, но и сохранить лояльность клиента. Падение NPS, с которого начался кейс, — это прямое следствие того, что жалобы обрабатываются некачественно.

---

## Что нужно построить

Диалоговый агент с чётким Tone of Voice, логикой оценки обоснованности жалобы и механизмом создания заявки в QA.

### Входные данные

```json
{
  "client_type": "B2C",
  "category": "complaint",
  "message": "Уже третий день не могу войти в приложение, никто не отвечает! Верните деньги за подписку!",
  "client_id": "u_456789",
  "dialog_history": [],
  "subscription_active": true,
  "days_since_issue": 3
}
```

### Выходные данные

```json
{
  "response_to_client": "Мне очень жаль, что вы столкнулись с такой ситуацией — три дня без доступа к приложению это действительно неприятно. Я уже передаю ваш запрос на проверку технической команде. В качестве компенсации за неудобства мы готовы продлить вашу подписку на 7 дней бесплатно. Ожидайте — мы свяжемся с вами в течение 2 часов.",
```

## Tone of Voice (обязательный)

### Принципы

1. **Эмпатия первым делом** — начать с признания проблемы клиента
2. **Конкретика** — не общие фразы, а конкретные действия и сроки
3. **Закрытие** — завершить диалог с чётким следующим шагом

### Фразы-триггеры (обязательны)

| Ситуация | Формула |
|----------|---------|
| Открытие жалобы | «Мне очень жаль, что вы столкнулись с такой ситуацией» |
| Признание проблемы | «Спасибо, что рассказали нам об этом. Давайте разберёмся» |
| Предложение компенсации | «В качестве извинения мы можем предложить вам...» |
| Передача в работу | «Я уже передаю ваш запрос...» |

### Запрещённые фразы

| Фраза | Почему нельзя |
|-------|--------------|
| «Это не наша вина» | Перекладывает ответственность, усиливает негатив |
| «Такова политика компании» | Звучит бездушно, закрывает диалог |
| «Прочитайте FAQ» | Пренебрежение к обращению клиента |
| «Ничем не могу помочь» | Недопустимо в поддержке |

---

## Логика оценки обоснованности

| Условие | Результат |
|---------|-----------|
| Проблема подтверждена технически | `justified: true` → компенсация по скрипту |
| Нет данных о проблеме | `justified: unclear` → уточняющие вопросы |
| Жалоба явно неправомерна | `justified: false` → вежливое разъяснение без компенсации |

### Скрипт компенсаций (прототип)

| Тип проблемы | Компенсация |
|--------------|-------------|
| Недоступность приложения > 1 дня | +7 дней подписки |
| Технический сбой при платеже | Возврат комиссии |
| Долгое ожидание ответа > 24ч | +3 дня подписки или промокод |
| Ошибка на стороне банка | Полный возврат суммы |

## Заявка в QA

При каждой обоснованной жалобе агент создаёт тикет:

```json
{
  "ticket_id": "QA-<дата>-<порядковый номер>",
  "priority": "High",
  "category": "<категория жалобы>",
  "client_id": "<id клиента>",
  "description": "<суть проблемы одним абзацем>",
  "compensation_issued": true
}
```

Эти тикеты передаются в **Задачу 7 (Аналитик)** для формирования отчётов.

## Тестовые сценарии

| Сообщение клиента | Ожидаемое поведение |
|-------------------|---------------------|
| «Уже третий день не могу войти в приложение, верните деньги!» | Эмпатия, компенсация (+7 дней), тикет в QA |
| «Сняли деньги дважды за одну операцию!» | Признание ошибки, возврат суммы, тикет Critical |
| «Приложение работает медленно» | Уточнение, `justified: unclear`, без компенсации |
| «Вы мошенники!» (необоснованно) | Эмпатия, разъяснение, `justified: false` |
| «Соедините с живым человеком» | `escalate_to_human: true`, прощальная фраза |

---

## Критерий приёма прототипа

- Агент отвечает на жалобу «Уже третий день не могу войти...» в правильном ToV
- Ни одна из запрещённых фраз не используется
- Компенсация предложена в соответствии со скриптом
- Тикет в QA создан с корректными полями
