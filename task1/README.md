# Задача 1: Входной классификатор и роутер

**Исполнитель:** Александр  
**Инструмент:** Evo Agents  
**Приоритет прототипа:** Высокий — точка входа всей системы

---

## Роль в системе

Александр отвечает за **входной классификатор** — первый узел цепочки агентов. Каждый входящий запрос клиента проходит через этот модуль до того, как попасть к любому другому агенту.

От качества классификации зависит корректность работы всей системы: неверно определённый тип клиента или категория запроса приведут к тому, что запрос попадёт не к тому агенту.

---

## Что нужно построить

Агент, который принимает на вход сырой текст клиентского сообщения и возвращает структурированный объект маршрутизации.

### Входные данные

```json
{
  "message": "Как настроить вебхук для уведомлений о платежах?",
  "source": "web_chat",
  "timestamp": "2026-04-28T09:15:00Z",
  "user_id": "u_123456"
}
```

### Выходные данные

```json
{
  "client_type": "B2B",
  "category": "technical_issue",
  "priority": "Normal",
  "route_to": "b2b_expert_agent",
  "confidence": 0.92,
  "summary": "Вопрос по настройке API/вебхук — корпоративный клиент"
}
```

---

## Логика классификации

### 1. Определение типа клиента (B2B / B2C)

| Признак | B2B | B2C |
|---------|-----|-----|
| Упоминание API, вебхук, интеграции | ✓ | |
| Упоминание ИНН, договора, счёта компании | ✓ | |
| Упоминание личного счёта, карты, приложения | | ✓ |
| Физическое лицо ("я", "мой счёт") | | ✓ |

### 2. Категория запроса

| Категория | Триггеры |
|-----------|----------|
| `consultation` | тариф, комиссия, условия, как работает |
| `complaint` | не работает, верните деньги, уже третий день, никто не отвечает |
| `security_incident` | взлом, украли, несанкционированный, не я, потерял доступ |
| `technical_issue` | API, вебхук, ошибка, не подключается, настройка |

### 3. Приоритет

| Приоритет | Условие |
|-----------|---------|
| `Critical` | Категория `security_incident` |
| `High` | Категория `complaint` или повторное обращение |
| `Normal` | Остальные случаи |

### 4. Маршрутизация

| Условие | Агент |
|---------|-------|
| B2C + consultation | `b2c_consultant_agent` |
| B2B + любая категория | `b2b_expert_agent` |
| security_incident | `security_ops_agent` |
| complaint | `complaints_agent` |

---

## Тестовые сценарии

| Входной текст | Ожидаемый результат |
|---------------|---------------------|
| «Какая комиссия за перевод?» | B2C, consultation, Normal, → b2c_consultant |
| «Как настроить вебхук?» | B2B, technical_issue, Normal, → b2b_expert |
| «Мне пришло смс о списании — это не я!» | B2C, security_incident, Critical, → security_ops |
| «Уже третий день не могу войти, верните деньги!» | B2C, complaint, High, → complaints |
| «Хватит ботов, соедините с менеджером!» | любой, escalation, High, → human |

---

## Интерфейс для состыковки с другими модулями

Выходной объект должен быть передан следующему агенту в виде контекста. Поля `client_type`, `category`, `priority` и `route_to` обязательны для всех нижестоящих агентов.
